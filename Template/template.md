# template.cppの説明

## #include<bits/stdc++.h>
C++で基本的な機能やデータ構造、STLなどを使用するために必要なファイルをまとめてダウンロードします。
環境によっては使えないこともあるそうですが、AtCoderとcodeforcesでは使えることを確認しています。

## using namespace std
本来cinやcoutを使うときは"std::cin"や"std::cout"と記述しなければなりませんが、これを書くことで"std::"を省略できます。

## using ll/ull/P/PIL/PLI/PL
long longを略してllと記述できます。ほかも同様に略して記述できるというだけです。unsignedとは、0以上の整数のみを扱うことで普通のlong longよりも大きな値を表現できます。10^19が出てくる場合llではオーバーフローしてしまいますが、ullではギリギリ表現できます。

## const ABC/abc/mod/INF/INFL
ABC/abcは文字列系の問題に対して、ある文字がアルファベットの何番目かを求める関数（後に記述があります）のために使います。 \
modはたまにmod998244353で求めてくださいという問題のときにいちいち数字を書かなくていいためのものです。少し古い問題では1000000007で求めてくださいという問題もありますが、ここを書き換えるだけで十分です。 \
INF/INFLは十分大きい数で初期化したいときなどに使います。実は使う機会はそこそこあります。

## const dx/dy
幅優先探索で上下左右に移動するときいちいち４方向の移動について書くと非常に面倒なので、現在の位置に対してdx[i],dy[i]を足すだけで上下左右の位置を表現できます。ループで0から3まで回しましょう。

## #define rep0/rep1/rrep0/rrep1
いわゆるrepマクロと言われるものです。rep0は0からN-1まで、rep1は1からNまで、rrep0はN-1から0まで、rrep1はNから1までのfor文を短く表現できます。

## #define el
"cout << ans << endl;"と書いたとき"<< endl"は最後に改行することを表しますが、「<< "/n"」と書くと計算量が少し良くなります。ただ、自分は"endl"で書くのに慣れてしまったので"el"を使うことで楽をしています。

## #define coutY/coutN
競プロではよく「判定してYesかNoかを出力せよ」という問題が出題されます。こういう問題では最後にある条件を満たせばYes、そうでなければNoとすることが多いので"(条件式) ? coutY : coutN;"と書くことで少し楽ができます。右の書き方を知らない方はAPG4bの3.06を参照してください。

## #define all(x)
vectorのsortやreverseのときに使います。vectorのsortは結構出てきます。上のcoutY/coutNもそうですが、そこまで大幅な短縮になるわけではなく個人的にコードがすっきりするので入れているだけです。

## template chmin/chmax
chminの場合、a>bならばa=bに変更しtrueを返す、a>bでないならばfalseを返します。chmaxの場合は条件式の不等号が逆になります。多くの競プロerが入れているであろうテンプレートです。DPなどで便利です。
